# DeepSeek Monitor — PySide6 暗色毛玻璃 UI 全面重设计计划

## 概述

将现有 tkinter UI 迁移至 PySide6，采用现代暗色毛玻璃（Glassmorphism）视觉风格，全面重新设计界面布局与交互。

**选定方案：**
- 框架：PySide6（LGPL 开源许可）
- 风格：现代暗色毛玻璃
- 范围：整体重设计

---

## 模块影响分析

| 模块 | 当前依赖 | 迁移后 | 说明 |
|------|----------|--------|------|
| `src/api.py` | 无 GUI | **不变** | HTTP 客户端 |
| `src/config.py` | 无 GUI | **不变** | 配置管理 |
| `src/tray.py` | 纯 Win32 API | **不变** | 系统托盘 |
| `src/taskbar_widget.py` | 纯 Win32 API | **不变** | 任务栏嵌入组件 |
| `src/app.py` | tkinter | **完全重写** → PySide6 | 主界面 |
| `main.py` | 无直接 GUI | **小改** | 窗口类名 + 启动方式 |
| `requirements.txt` | Pillow | **添加 PySide6** | 新依赖 |
| `tests/` | 无 GUI 依赖 | **不变** | 后端测试 |

## 新建文件

| 文件 | 职责 |
|------|------|
| `src/theme.py` | 全局颜色常量、字体常量、QSS 样式表 |
| `src/widgets.py` | 可复用组件：GlassCard、StatCard、CollapsibleSection、RangeSelector |
| `src/charts.py` | 自定义余额趋势图（QPainter 绘制） |

---

## 实施阶段

### 阶段 1：依赖更新与入口适配

- 在 `requirements.txt` 添加 `PySide6>=6.5`
- 修改 `main.py`：
  - 单实例检测中的窗口类名从 `"TkTopLevel"` 改为 PySide6 类名（`"Qt64Window"` 或按窗口标题 `"DeepSeek Monitor"` 查找，已有按标题查找的兜底逻辑）
  - `main()` 函数中将 `DeepSeekMonitorApp.run()` 改为 PySide6 启动模式：`QApplication` → `DeepSeekMonitorApp(root, config)` → `app.exec()`
- 验证：程序能正常启动（即使 `src/app.py` 还未重写，先保留一个最小 Qt 窗口占位）

### 阶段 2：主题系统（`src/theme.py`）

- 定义颜色常量：
  - 背景主色 `#1a1a2e`、次色 `#16213e`
  - 卡片背景 `rgba(255, 255, 255, 0.05)`、边框 `rgba(255, 255, 255, 0.08)`
  - 强调色 `#7c3aed`（紫色）、成功 `#22c55e`、警告 `#eab308`、危险 `#ef4444`
  - 文字主色 `#e2e8f0`、次色 `#94a3b8`
- 定义字体常量：`title=22pt bold`、`balance=36pt Consolas bold`、`section=13pt bold`、`body=10pt`、`mono=10pt Consolas`
- 编写全局 QSS 样式表，覆盖：
  - `QMainWindow` 背景色
  - `QTabWidget::pane` / `QTabBar::tab` 选中态强调色背景
  - `QPushButton` 圆角 8px + hover 发光效果
  - `QLineEdit` / `QComboBox` 暗色输入框 + 圆角
  - `QCheckBox` 自定义样式
  - `QScrollArea` 透明滚动条
  - `QFrame`（卡片）圆角 + 微光边框
- 验证：在一个空窗口中应用样式表，确认视觉效果

### 阶段 3：可复用组件（`src/widgets.py`）

- **`GlassCard(QFrame)`**：半透明背景 + 1px 微光边框 + 12px 圆角，可选标题
- **`StatCard(GlassCard)`**：上方小号标签 + 下方大号数值，支持颜色随数据变化（绿/黄/红）
- **`CollapsibleSection(QWidget)`**：点击标题行展开/折叠，右侧显示状态摘要（如 "已启用 ✓"）
- **`RangeSelector(QWidget)`**：QButtonGroup 圆角 toggle 按钮组，替代 tk.Button 横排
- 验证：在测试窗口中展示各组件，确认交互正常

### 阶段 4：图表组件（`src/charts.py`）

- **`BalanceChart(QGraphicsView)`**：使用 `QPainter` 自绘
  - 折线 + 下方渐变填充区域（area chart）
  - 鼠标悬停 tooltip 显示精确时间与余额值
  - 标注最大值/最小值点（圆点 + 标签）
  - Y 轴 5 档网格线（虚线）+ 金额标签
  - X 轴自适应标签密度
  - 窗口缩放自动重绘
- 验证：用模拟数据渲染图表，确认缩放、tooltip、min/max 标注正常

### 阶段 5：主窗口与页面重写（`src/app.py`）

窗口结构：

```
QMainWindow (920×640)
├── Header (QWidget)
│   ├── 第一行: 标题 + 刷新按钮 + 状态时间
│   └── 第二行: 4×StatCard（总余额 / 状态 / 赠送 / 充值）
├── QTabWidget
│   ├── Tab 1: 📊 花费趋势（提升为首个标签）
│   │   ├── RangeSelector（日/周/月/季度/年）
│   │   ├── BalanceChart
│   │   └── 统计摘要栏（起始 / 当前 / 变化）
│   ├── Tab 2: 📊 概览
│   │   ├── 2×2 StatCard 矩阵（总余额 / 今日花费 / 赠送 / 充值）
│   │   ├── 预计可用天数卡片
│   │   └── 模型定价速查表
│   └── Tab 3: ⚙ 设置
│       ├── CollapsibleSection: API Key
│       ├── CollapsibleSection: 刷新间隔
│       ├── CollapsibleSection: 任务栏小组件（默认折叠）
│       ├── CollapsibleSection: 语言
│       ├── CollapsibleSection: 开机自启
│       └── CollapsibleSection: 关于
└── 系统托盘（复用 tray.py）
```

具体实施步骤：

- 重写 `DeepSeekMonitorApp` 类，使用 `QMainWindow` 替代 `tk.Tk`
- 实现 Header：4 个 `StatCard` 横向排列，`balance` 数值随 API 数据变色
- 实现 Tab 1（花费趋势）：`RangeSelector` + `BalanceChart` + 统计摘要
  - 新增"今日花费"计算：从 `balance_history_day.json` 取首尾差值
  - 新增"预计可用天数"计算：近 7 天日均花费 → 余额 / 日均
- 实现 Tab 2（概览）：2×2 `StatCard` + 预计天数 + 定价表
- 实现 Tab 3（设置）：6 个 `CollapsibleSection`，任务栏配置默认折叠
- 数据刷新统一使用 `QTimer`（替代 `root.after`）
- 中英双语保持现有 `T` 字典方案，语言切换时遍历 widget 更新文本
- 系统托盘调用保持不变（`tray.py` 无需修改）
